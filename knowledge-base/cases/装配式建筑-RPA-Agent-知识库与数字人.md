# 用户案例：某装配式建筑制造企业 Astron 知识库与数字人

*信息来源：项目沟通记录及用户反馈 (2025-12-22 ~ 2026-01-22)*

---

## 1. 案例概况 (Case Overview)
*   **项目名称**：某装配式建筑制造企业 Astron 部署与应用项目
*   **客户方**：某装配式建筑制造企业
*   **应用产品**：Astron Agent, Astron RPA
*   **部署状态**：Astron Agent (测试环境/内测中), Astron RPA (开发环境/联调中)
*   **核心场景**：企业内部知识库、数字人导览及智能体应用

## 2. 业务背景 (Background)
某装配式建筑制造企业是一家专注于装配式建筑及制造业的企业。为了提升企业内部知识管理效率及对外展示效果，客户计划引入 AI 技术：
*   **内部知识库**：构建基于企业私有数据的问答系统，涵盖模具设计、装配式建筑安装工艺、风电混塔技术参数等专业内容，用于员工专业咨询与解答。
*   **数字人导览**：在企业展厅部署虚拟数字人，为访客提供产品介绍与自动导览服务，并希望结合大屏背景进行沉浸式讲解。

## 3. 挑战与痛点 (Challenges)
在私有化部署与初期调试过程中，技术团队遇到了以下关键问题：
*   **容器化部署不稳定**：初期部署时出现核心服务（如 `astron-agent-casdoor`）反复重启、容器无法正常启动或日志缺失的问题，导致服务不可用。
*   **网络与认证配置复杂**：在配置 HTTPS 访问及 Nginx 反向代理时，频繁遇到“自动跳转登录页死循环”、“502 Bad Gateway”以及 Casdoor 认证服务的 `redirect_uri` 不匹配错误，导致用户无法正常登录系统。
*   **环境依赖冲突**：Docker 环境中的端口映射冲突（如 80 端口被占用），以及前后端服务地址配置（`VITE_API_URL` 与 `API_URL`）不一致，导致前端无法正确调用后端 API。
*   **业务场景定制需求**：客户对“数字人”有特定的定制需求（如隐藏对话框、自定义背景），标准版功能需进一步适配或二次开发。

## 4. 解决方案 (Resolution)
Astron 技术支持团队通过群协同提供了针对性的排查与修复方案：
1.  **环境重置与标准化部署**：指导客户清理旧的 Docker 容器与数据卷（Volumes），重新拉取最新镜像，并严格按照标准模板配置 `.env` 环境变量，解决了服务启动不稳定的问题。
2.  **网络配置调优**：
    *   协助修改 `nginx.conf`，正确配置 WebSocket 支持及 HTTPS 证书路径，解决 502 报错。
    *   校准 `docker-compose.yaml` 中的服务地址配置，确保后端 API 地址与外部访问域名一致。
    *   在 Casdoor 管理后台正确添加回调地址（Redirect URIs），修复了登录重定向错误。
3.  **端口与服务映射**：调整宿主机端口映射，避开本地端口冲突；明确 Agent 与 RPA 的联合部署架构，确保各组件通信正常。
4.  **场景化建议**：针对数字人导览需求，推荐使用“星火快答”解决方案或基于 Astron 开源项目进行前端二次开发，以满足定制化的 UI 展示需求。

## 5. 客户价值 (Value)
*   **成功构建私有知识库底座**：协助客户在内网环境成功跑通 Astron Agent 与 RPA 平台，为后续导入企业专业文档（如建筑工艺、技术参数）奠定了基础。
*   **打通自动化与 AI 链路**：解决了环境搭建与网络配置的“拦路虎”，使客户能够进入实际业务场景（如知识库问答、智能体编排）的验证阶段。
*   **明确技术演进方向**：通过对特定需求（数字人）的技术咨询，为客户明确了“标准功能+二次开发”的落地路径。
