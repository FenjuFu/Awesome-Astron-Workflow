# 用户案例：某大型能源企业 Astron Agent 第三方大模型接入

*信息来源：项目沟通记录 (2026-03-09 ~ 2026-03-10)*

---

## 1. 案例概况 (Case Overview)
*   **项目名称**：某大型能源企业 Astron 项目
*   **客户方**：某大型能源企业
*   **应用产品**：Astron Agent
*   **部署状态**：生产环境
*   **核心场景**：模型管理、第三方大模型接入

## 2. 业务背景 (Background)
某大型能源企业在使用 Astron Agent 进行智能化建设，旨在通过接入外部强大的第三方大模型能力，提升业务流程的自动化与智能化水平。

## 3. 挑战与痛点 (Challenges)
*   **模型配置长度限制**：在系统“模型管理”模块尝试关联第三方模型时，用户发现系统前端对“接口地址 (Interface Address)”和“密钥 (API Key)”的输入框存在硬性字符限制（100 字符）。由于客户所使用的第三方模型服务提供的接口 URL 和鉴权 Key 长度超过了此限制，导致无法在界面上完成配置，阻碍了模型的接入与使用。

## 4. 解决方案 (Resolution)
针对内网部署环境下的这一限制，Astron 技术支持团队提供了临时的源码修改方案，帮助客户突破限制：

1.  **前端源码调整**：
    *   **定位文件**：`console/frontend/src/pages/model-management/components/modal-component.tsx`
    *   **修改参数**：在该文件中找到 `interfaceAddress` 和 `apiKEY` 对应的输入框配置，将 `maxLength={100}` 属性修改为 `maxLength={500}`（或根据实际需求调整为更大的值）。
2.  **重构与部署**：
    *   在修改完成后，执行前端构建命令（如 `npm run build`）重新生成静态资源。
    *   将新的构建产物部署至服务器，并刷新页面或重启服务，使配置生效。

## 5. 客户价值 (Value)
*   **解决兼容性问题**：通过灵活的技术支持方案，解决了产品默认配置无法适配特定第三方服务长 URL/Key 的问题。
*   **确保能力接入**：保障了客户能够顺利集成目标大模型，为后续的 AI 业务场景落地奠定了基础。

## 6. 产品共创与反馈 (Co-creation & Feedback)
*   **配置限制优化**：建议产品研发团队在后续版本中评估并放宽模型管理中相关字段的长度限制，或采用更动态的验证机制，以直接适配更多类型的第三方模型服务，减少用户手动修改源码的运维成本。
