# 用户案例：某智慧园区数字科技公司 Astron Agent 数据分析

*信息来源：项目沟通记录及用户反馈 (2025-12-02 ~ 2026-01-28)*

## 1. 案例概况 (Case Overview)
*   **项目名称**：某智慧园区数字科技公司智慧园区客服与数据分析项目
*   **客户方**：某智慧园区数字科技公司
*   **应用产品**：Astron Agent
*   **部署状态**：生产环境（全量上线）
*   **核心场景**：智慧园区客服、竞品/热点数据分析、非结构化数据清洗
*   **后续需求**：API 能力扩展（支持高阶配置调用）、MCP 协议支持、数据安全审计、智能体观测

## 2. 业务背景 (Background)
某智慧园区数字科技公司致力于利用 AI 技术赋能智慧园区运营。企业希望构建一套智能系统，一方面能够抓取全网竞品评论与热点趋势，提炼用户痛点与爆款选题；另一方面能够分析客户画像与行为，自动发送个性化问候、产品推荐与售后关怀。此外，业务中存在大量非结构化文本数据（如复杂的订单信息）需要转换为结构化数据录入系统。

## 3. 挑战与痛点 (Challenges)
在项目落地过程中，团队主要面临以下技术挑战：
*   **非结构化数据处理**：业务场景中包含复杂的非标准文本（如：“1 土豆2件 公斤 5...”），传统的规则提取器难以准确处理，导致无法自动化录入系统。
*   **部署环境受限**：在私有化部署过程中，遇到 Docker 镜像拉取失败、服务无法访问等环境配置问题。
*   **多模型兼容性**：需要集成腾讯等特定厂商的大模型服务，对平台的兼容性提出了要求。
*   **API 能力限制**：当前 API 接口仅支持调用工作流，暂不支持调用包含“高阶配置”的智能体，限制了复杂业务场景的自动化联动。
*   **开源版功能差异**：开源版本工具节点暂不支持自定义 MCP（Model Context Protocol）节点，且部分高阶配置无法保存，影响了特定功能的扩展与调试。

## 4. 解决方案 (Resolution)
Astron 技术支持团队协助客户完成了以下建设：
1.  **智能解析工作流**：针对非结构化数据，采用 **LLM（大模型）+ Prompt** 的方式替代传统提取器。通过精心设计的提示词，将复杂的文本信息精准转化为标准 JSON 格式，随后调用 API 自动写入业务系统。
2.  **环境配置优化**：提供国内镜像源（如 `ghcr.nju.edu.cn`）解决 Docker 部署网络问题，确保生产环境顺利上线。
3.  **标准化模型接入**：基于 Astron 平台的 OpenAI-compatible 协议支持，成功接入腾讯大模型，满足了客户对特定底层模型的需求。

## 5. 客户价值 (Value)
*   **市场洞察自动化**：实现了全网数据的自动抓取与分析，帮助企业快速捕捉市场热点与用户痛点。
*   **服务体验升级**：基于客户画像的自动化关怀与推荐，显著提升了园区服务的个性化水平与响应速度。
*   **业务流程闭环**：打通了从非结构化文本到业务系统数据的自动化链路，大幅减少了人工录入成本与错误率。
