# 用户案例：某产业互联网企业 Astron Agent 企微集成

*信息来源：项目沟通记录 (2026-03-04 ~ 2026-03-06)*

---

## 1. 案例概况 (Case Overview)

*   **项目名称**：某产业互联网企业Agent应用与集成
*   **客户方**：某产业互联网企业
*   **应用产品**：Astron Agent
*   **部署状态**：开发环境（二开/评估中）
*   **核心场景**：企微集成、Agent规划能力、无视觉模型环境

## 2. 业务背景 (Background)
某产业互联网企业正在探索使用 Astron Agent 进行二次开发，重点关注企业微信集成（扫码登录、SSO）、无视觉模型环境下的运行以及具备规划能力的智能体应用。

## 3. 挑战与痛点 (Challenges)
在探索与二开过程中，技术团队遇到了以下需求与问题：

1.  **企业微信集成需求**：
    *   **登录方式**：希望实现企业微信扫码登录。
    *   **单点登录 (SSO)**：需要支持企业微信单点登录功能。
    *   **二开咨询**：咨询是自行二开实现还是等待官方迭代支持。

2.  **多模态与规划能力**：
    *   **无视觉模型**：提出了“无视觉模型”的场景需求（具体背景需确认，可能是资源受限或特定业务场景）。
    *   **Agent规划**：需要“带有规划的智能体”能力，期望提升Agent的任务拆解与执行能力。

3.  **多租户/组织登录问题**：
    *   **登录失败**：用户反馈特定组织的账号无法登录后台（`http://127.0.0.1:8000/`）。
    *   **配置异常**：经排查，疑似 Casdoor 配置存在问题，导致非 built-in 组织用户无法正常认证。

## 4. 解决方案 (Resolution)
Astron 技术支持团队提供了以下支持与建议：

1.  **登录问题排查**：
    *   **临时规避**：建议将用户（如 `momo`）移动到 `built-in` 组织下尝试登录。
    *   **问题定位**：确认为 Casdoor 配置不正确导致。
    *   **Issue 跟踪**：引导用户在 GitHub 提交 Issue 以便后续修复与跟踪。

2.  **需求反馈与共建**：
    *   **企微集成**：针对企微扫码登录与 SSO 需求，建议用户提交 Issue，视情况纳入官方路线图或由社区共建。
    *   **功能咨询**：针对无视觉模型和规划智能体需求，技术支持介入咨询具体场景。

## 5. 客户价值 (Value)
*   **明确二开方向**：明确了企微集成等关键特性的实现路径（社区共建/官方迭代）。
*   **问题快速定位**：快速定位了多租户/多组织下的登录配置问题，保障了基础环境的可用性。
*   **参与开源共建**：引导客户将具体需求转化为 Issue，促进了产品功能的完善。
