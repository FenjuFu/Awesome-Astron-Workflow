# Astron 智能体平台常见问题解答 (FAQ)

> 本文档基于官方交流群的问答记录整理归纳，涵盖部署、RPA、配置及功能使用等常见问题。

## 一、 部署与安装 (Deployment & Installation)

### Q: 部署后接口报错 502 Bad Gateway 怎么办？
**A:** 502 通常表示后端服务未正常响应。
1. 请检查所有容器的运行状态 (`docker ps`)，确认是否有容器退出或重启。
2. 查看相关服务容器的日志 (`docker logs <container_id>`) 寻找报错信息。

### Q: 宝塔面板部署后访问异常？
**A:** 
1. 确保项目容器已成功启动。
2. 尝试直接访问入口的默认端口（通常为 80）。
3. 检查防火墙或宝塔的安全组设置是否放行了相关端口。

### Q: Docker Compose 启动失败，提示找不到 `arigaio/atlas` 等镜像？
**A:** 可能是网络问题导致无法拉取国外镜像。
1. 检查服务器网络连接。
2. 配置 Docker 镜像加速器。
3. 尝试手动拉取该镜像。

### Q: 本地部署必须安装 Docker 吗？
**A:** 是的，Astron Agent 平台依赖 Docker 进行容器化部署，必须先安装 Docker 环境。

### Q: Casdoor 重启后数据丢失（如账号、配置消失）？
**A:** 
1. 检查启动命令，`docker compose down` 只是停止并移除容器，不会删除数据卷；但如果使用了 `down -v` 参数，则会**清空数据卷**导致数据丢失。
2. 检查 `docker-compose.yaml` 中关于 Casdoor 的 `volumes` 配置是否正确映射了本地路径以实现持久化。

### Q: 如何更新到最新版本？
**A:**
1. 拉取最新的代码或镜像。
2. 执行 `docker compose -f docker-compose-with-auth.yaml down` 停止服务（如果需要彻底重置环境且不保留数据，可加 `-v`，**慎用**）。
3. 如果修改了 git 追踪的文件，可执行 `git restore docker` 清理变更。
4. 执行 `docker compose -f docker-compose-with-auth.yaml up -d` 重新启动。
5. 建议清理浏览器缓存或使用无痕模式访问，避免前端缓存导致的问题。

### Q: 创建应用失败，日志显示 403 错误？
**A:** 403 通常是权限或认证问题。请检查环境变量配置（如 API Key、Secret 等）是否正确填写，且与部署文档要求一致。

## 二、 RPA 相关 (AstronRPA)

### Q: RPA 机器人构建或启动时提示 `Python复制失败`？
**A:** 这是一个常见问题，可能原因及解决方法如下：
1. **路径问题**：检查项目路径是否包含**中文**或特殊字符，建议移动到纯英文路径下重试。
2. **权限问题**：尝试以**管理员身份**运行 CMD 或终端。
3. **脚本问题**：检查 `build.bat`（约 180 行左右），尝试将 `xcopy` 命令修改为：
   `xcopy /E /I /Y "%PYTHON_SOURCE_DIR%\*" "%PYTHON_CORE_DIR%\"`
4. **版本问题**：确保使用的是较新的代码版本（建议拉取最新代码）。

### Q: RPA 依赖下载失败、卡住或报错？
**A:** 
1. **网络问题**：国内网络环境可能导致连接 PyPI 官方源超时。
2. **解决方案**：配置国内镜像源（如阿里云）。
   - 在 `pip` 配置中添加：
     ```ini
     [global]
     index-url = https://mirrors.aliyun.com/pypi/simple/
     trusted-host = mirrors.aliyun.com
     ```
   - 或者在安装命令中指定镜像源。

### Q: RPA 可以在内网使用吗？
**A:** 可以。部署好服务端后，RPA 客户端和服务端在同一内网环境下即可正常通信和使用。

### Q: 运行 RPA 任务报错 `Task exec failed: send uuid empty`？
**A:** 尝试重启 RPA 客户端及机器人服务。

### Q: 拾取元素时报错“拾取元素类型需要跟浏览器类型保持一致”？
**A:** 
1. 检查当前拾取工具选择的浏览器类型（如 Chrome、Edge）是否与实际操作的浏览器一致。
2. 如果你要点击的是浏览器本身的操作栏（如关闭按钮、地址栏）或者其他桌面软件，请使用**“桌面元素”**模式进行拾取，不要使用“浏览器元素”模式。

### Q: 虚拟桌面运行失败？
**A:** 部分操作系统环境可能不支持虚拟桌面功能，建议在标准桌面环境下测试。

## 三、 配置与环境 (Configuration)

### Q: 修改了 IP 地址或端口配置后不生效？
**A:** 修改 `.env` 文件或 `docker-compose.yaml` 中的环境变量后，必须重启容器才能生效：
`docker compose down` 然后 `docker compose up -d`。

### Q: 找不到 `.env` 文件在哪里？
**A:** `.env` 文件通常位于项目的根目录下。如果看不到，可能是文件被隐藏，或者需要从 `.env.example` 复制并重命名。

### Q: 验证码接口报错或无法登录？
**A:** 
1. 检查 Casdoor 服务是否正常启动。
2. 检查验证码相关配置是否正确。
3. 如果是测试环境，可尝试使用默认管理员账号 `admin` / `123` 登录。

## 四、 功能与使用 (Features & Usage)

### Q: 开源版和企业版有什么区别？
**A:** 开源版提供核心的基础功能供开发者学习和使用；企业版通常包含更高级的企业级特性，如更细粒度的权限管理（RBAC）、更强的审计、性能优化及原厂支持等。

### Q: 支持自定义 MCP (Model Context Protocol) 工具吗？
**A:** 支持。可以在 Web 端的工作流节点（如 Agent 智能决策节点）中添加和配置 MCP 工具。

### Q: 知识库（RAG）引用有问题，无法检索或回答？
**A:** 
1. 早期版本的对话型 Agent 在引用知识库时可能存在 Bug，建议更新到最新版本的镜像。
2. 目前系统使用的是 RAGFlow (v0.12.0 或指定版本) 作为底层引擎。
3. **工作流模式**下引用知识库通常更稳定。

### Q: 知识库如何控制模型回答（防止幻觉）？
**A:** 
1. 检索到的知识库内容会作为上下文填充到 Prompt 中发送给模型。
2. 可以通过修改提示词（Prompt）来约束模型：例如添加“请仅依据检索到的内容回答，如果检索内容中没有答案，请直接回复不知道，不要编造”。

### Q: 如何自定义原子组件？
**A:** 目前流程可能需要修改代码后，手动将新的原子树信息同步/更新到数据库中。后续版本计划提供更便捷的自定义组件开发方式。
